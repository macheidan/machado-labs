The problem: There are many simultaneous projects, decisions made in different contexts, and every AI agent starts from scratch knowing nothing about me.

The solution I found: build an "external brain" in markdown that serves as the single source of truth for any AI agent.

## The core idea

I don't want to depend on a specific tool. I want simple `.md` files, version-controlled in git, synced through Google Drive, and readable by any model — Claude, ChatGPT, Gemini, Cursor, whatever.

The vault lives at `G:\My Drive\01 AI\Vault\` and is already open in Obsidian. Any agent that needs context about me reads the files from there.

## Architecture in 4 layers

```
┌─────────────────────────────┐
│ 4. TRIGGERS (cron, webhook) │
├─────────────────────────────┤
│ 3. AGENCY (Gmail, Cal, WA)  │  ← I'm here
├─────────────────────────────┤
│ 2. PERSONA (how I decide)   │  ← done
├─────────────────────────────┤
│ 1. MEMORY (what I know)     │  ← done
└─────────────────────────────┘
```

## Phase 1 — Memory

I swept every source: Google Drive folders, dev projects, memories auto-generated by Claude Code, scattered context files. I classified everything into categories and copied it into the vault structure — never moving originals without approval.

The structure ended up like this:

```
Vault/
├── 00-meta/           # instructions for any agent
├── 01-pizzerias/      # Dáme and Lov — marketing, ops, finance
├── 02-dev-projects/   # all code projects
├── 03-personal/       # family, health, finances, profile
├── 04-consulting/     # projects and clients
├── 08-secretary-log/  # operational logs (gitignored)
```

Every file has YAML frontmatter with `project`, `status`, `updated`, and `tags`. That lets any agent understand the context without having to read the entire file.

## Phase 2 — Persona

I captured how I actually make decisions — not how I think I make them, but how I really do. That became `persona-decision.md`:

- I decide alone, rarely asking for input first
- I prefer data and numbers, but impulse can override the analysis
- Two modes: fast/reactive and analytical (when I force it)
- Tendency to overestimate familiarity with a problem and underestimate real effort

The file isn't aspirational. It's descriptive. The agent uses it to flag me when it notices I'm in the wrong mode for an important decision.

## Phase 3 — Agency (in progress)

Here the agents stop being advisory and start acting. First I integrated WhatsApp.

The bot runs in the background via PM2 — no Chrome, no open window. Every week I get on my own WhatsApp:

- Behavioral profiles of the contacts I've talked to most
- A summary of group debates
- Analysis of my own messages — what the bot observed about me that could enrich the vault

The most interesting part: every Sunday the bot reads my `profile-fabio.md`, `persona-decision.md`, and `family.md` files and adds only what's genuinely new — without overwriting, without inventing, only what it could observe in the week's messages.

## What I've learned so far

The vault only works if it's easy to maintain. Complicated files don't get read, don't get updated, and die. Pure markdown, simple frontmatter, no proprietary plugins.

The automation that feeds the vault has to be conservative — better to write nothing than to write the wrong thing. That's why the bot uses a prompt that instructs Gemini to return "nothing new" if there are no clear observations based on the messages.

## Next steps

Gmail and Google Calendar enter Phase 3 — important email summaries, appointment alerts, vault integration. Then comes Phase 4: automatic triggers that fire actions without me having to ask.

The end goal is a system that knows my context better than I do myself on any overloaded day.
